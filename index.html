# Privacy Policy Setup Guide This guide explains how to host and link your privacy policy for Google Play Console. ## Files Created 1. **PRIVACY_POLICY.html** - A styled HTML version of the privacy policy (ready to host) 2. **PRIVACY_POLICY.md** - A Markdown version (for GitHub or documentation) ## Hosting Options ### Option 1: Separate Public GitHub Repo (Recommended for Private Main Repo) Since your main repo is private, create a separate **public** repository just for the privacy policy: 1. Create a new public GitHub repository (e.g., `expenza-privacy-policy`) 2. Upload `PRIVACY_POLICY.html` to the repository (rename to `index.html` for cleaner URL) 3. Enable GitHub Pages: - Go to repository Settings → Pages - Select source branch (usually `main` or `master`) - Save 4. Your privacy policy will be available at: - `https://[your-username].github.io/expenza-privacy-policy/` - This keeps your main code private while making the privacy policy public **Quick Setup:** ```bash # Create a new directory for the privacy policy repo mkdir expenza-privacy-policy cd expenza-privacy-policy cp /path/to/PRIVACY_POLICY.html index.html git init git add index.html git commit -m "Add privacy policy" git branch -M main gh repo create expenza-privacy-policy --public git remote add origin https://github.com/yourusername/expenza-privacy-policy.git git push -u origin main # Then enable GitHub Pages in repo settings ``` ### Option 2: Netlify (Free & Easy) 1. Go to [Netlify](https://www.netlify.com/) and sign up (free) 2. Drag and drop the folder containing `PRIVACY_POLICY.html` (or use Netlify CLI) 3. Your privacy policy will be available at: `https://[random-name].netlify.app/PRIVACY_POLICY.html` 4. You can add a custom domain later if needed **Netlify CLI Setup:** ```bash # Install Netlify CLI: npm install -g netlify-cli netlify login netlify deploy --prod --dir . --file PRIVACY_POLICY.html ``` ### Option 3: Vercel (Free & Easy) 1. Go to [Vercel](https://vercel.com/) and sign up (free) 2. Create a new project and upload `PRIVACY_POLICY.html` 3. Your privacy policy will be available at: `https://[project-name].vercel.app/PRIVACY_POLICY.html` ### Option 4: GitHub Gist (Simple but Limited) 1. Go to [GitHub Gist](https://gist.github.com/) 2. Create a new gist with `PRIVACY_POLICY.html` content 3. Make it public 4. Use the raw URL: `https://gist.githubusercontent.com/[username]/[gist-id]/raw/PRIVACY_POLICY.html` - Note: This works but the URL is not very clean ### Option 5: Your Own Website 1. Upload `PRIVACY_POLICY.html` to your web server 2. Access it via: `https://yourdomain.com/privacy-policy.html` ### Option 6: Privacy Policy Generator Services - [Privacy Policy Generator](https://www.privacypolicygenerator.info/) - [FreePrivacyPolicy](https://www.freeprivacypolicy.com/) - [Termly](https://termly.io/) ## Adding to Google Play Console 1. Go to [Google Play Console](https://play.google.com/console) 2. Select your app (Expenza: Expense Tracker) 3. Navigate to: **Policy** → **App content** 4. Scroll to **Privacy Policy** section 5. Click **Start** or **Manage** 6. Enter your privacy policy URL (e.g., `https://yourusername.github.io/yourrepo/PRIVACY_POLICY.html`) 7. Click **Save** ## Important Notes 1. **Update Contact Email:** Before publishing, replace `[Your Contact Email]` in the privacy policy with your actual contact email address. 2. **Accessibility:** Make sure the privacy policy URL is: - Publicly accessible (no login required) - Uses HTTPS (required by Google Play) - Returns a 200 OK status code 3. **Content Updates:** If you update the privacy policy, update the "Last Updated" date in the file. 4. **Testing:** Test the URL in an incognito/private browser window to ensure it's publicly accessible. ## Quick Checklist - [ ] Privacy policy hosted and accessible via HTTPS - [ ] Contact email updated in privacy policy - [ ] Privacy policy URL added to Google Play Console - [ ] Privacy policy URL tested in incognito mode - [ ] "Last Updated" date is current ## Recommended: Separate Public Repo for Privacy Policy Since your main repo is private, create a separate public repository: ```bash # Create a new directory mkdir expenza-privacy-policy cd expenza-privacy-policy # Copy and rename the privacy policy cp /path/to/PRIVACY_POLICY.html index.html # Initialize git git init git add index.html git commit -m "Add privacy policy for Expenza app" # Create public repo (using GitHub CLI) gh repo create expenza-privacy-policy --public --source=. --remote=origin git push -u origin main # Or manually: # 1. Create a new public repo on GitHub named "expenza-privacy-policy" # 2. Then: git remote add origin https://github.com/yourusername/expenza-privacy-policy.git # 3. git push -u origin main ``` Then enable GitHub Pages: 1. Go to the repository on GitHub 2. Settings → Pages 3. Select branch: `main` and folder: `/ (root)` 4. Save Your privacy policy URL will be: `https://yourusername.github.io/expenza-privacy-policy/` This keeps your main codebase private while making the privacy policy publicly accessible.
